TimeMachine time_delay 슬라이딩 윈도우

time_delay 파라미터 값은 항상 MIN~MAX 범위 내. 콘텐츠 윈도우는 시간이 흐르면서 자동 슬라이딩.

타임라인 시각화

1. time_delay 파라미터 값 범위 (API 호출 시 실제 사용되는 값)

X축 = 테스트 시작 후 경과시간 | Y축 = time_delay 값 (몇 초 전 콘텐츠를 요청하는지) | Phase 1: 성장 → Phase 2: MIN~MAX 고정

2. 실제 콘텐츠 윈도우 (테스트 시작 기준 절대 시간)

X축 = 테스트 경과시간 | Y축 = 콘텐츠 절대 위치 (테스트 시작=0초) | Phase 2에서 delay 값은 같지만 콘텐츠 구간이 자동 슬라이딩

3. 캐시키 수 변화 (렌디션 14개 x time_delay 범위)

Phase 1에서 증가, Phase 2에서 고정 (슬라이딩이므로 새 키 생성 + 오래된 키 만료 = 총 수 일정)
